# 次に読むべき論文 (Next to Read)

GEPA (Genetic-Pareto) は2025年7月に公開され、ICLR 2026に採択されたばかりの非常に新しい論文です。そのため、**現時点では本論文を直接引用（Cited by）している後続の査読付き論文は見つかりませんでした**。

しかし、GEPAが取り組んでいる「テスト時（推論時）計算のスケーリング」や「強化学習（RL）や進化的アルゴリズムを用いたプロンプト最適化」は、2025年〜2026年におけるLLM研究の最大のトレンド（Inference-Time Scaling）となっています。
GEPAの代替となるアプローチや、同時期に発表された同分野の最新研究（Concurrent Work）として、以下の論文や手法を次に読むことを推奨します。

## 1. Multi-Agent Test-Time Reinforcement Learning (MATTRL)
- **概要**: 単一エージェントではなく、複数のエージェントが協調してテキストベースの経験を注入し合い、テスト時に強化学習を用いて戦略を洗練させる手法です。
- **GEPAとの関連**: GEPAは単一のオプティマイザがパレートフロントから探索を行いますが、MATTRLはマルチエージェントの相互作用をテスト時RLの枠組みに落とし込んでおり、進化・改善の別のアプローチとして比較検討する価値があります。

## 2. RATTPO (Reward-Agnostic Test-Time Prompt Optimization)
- **概要**: タスク固有の報酬（Reward）設計を必要とせずに、多様なシナリオにわたってプロンプトを動的に最適化するフレームワークです。
- **GEPAとの関連**: GEPAは各モジュールのフィードバック関数（$\mu_f$）を設計してリフレクションに利用しますが、RATTPOは報酬関数を設計する手間を省きながら同等のテスト時最適化を目指す点で、実世界のより曖昧なタスクへの適用においてシナジーまたは対比となる研究です。

## 3. Test-Time Policy Adaptation for Multi-Turn Interactions (T2PAM)
- **概要**: 単発のプロンプト最適化ではなく、ユーザーとのリアルタイムなマルチターン対話のフィードバックを報酬信号として用い、セッション中にモデルのポリシーをテスト時適応（Test-Time Adaptation）させる手法です。
- **GEPAとの関連**: GEPAはシステム構築（あるいはプロンプト設計）のオフライン・インザループな最適化に強みを持ちますが、T2PAMはより動的でオンラインな対話へのテスト時適応に焦点を当てており、「対話を通じた継続的な改善」という観点で発展的なアイディアを提供します。
