# 次に読むべき関連論文 (EvoPromptの被引用・発展研究)

本論文「EvoPrompt」は、LLMを進化的アルゴリズム（EA）における「進化・変異オペレーター」として活用することで、プロンプトの自動最適化を実現し、探索（Exploration）と活用（Exploitation）のバランスを取ることに成功しました。

このアプローチをさらに発展させる、または関連してプロンプト最適化フレームワークを検証している最新の研究として、以下の論文を提案します（ハルシネーション排除のため、実際に取得された被引用リストから厳選しています）。

## 1. 進化プロセスのさらなる自己適応と最適化
*EvoPromptは固定の選択・変異プロセスを使用していましたが、進化のプロセス自体を最適化・動的調整する研究が登場しています。*

*   **EvoX: Meta-Evolution for Automated Discovery (2026)**
    *   概要: EvoPromptなどの手法は固定の探索戦略（explore-exploit比率など）に依存していますが、本研究「EvoX」は、進化プロセスにおける探索戦略そのものを動的に最適化する「メタ進化手法」を提案しています。最適化タスクにおける過去の結果に基づいて戦略を切り替えることで、多様なベンチマークで既存のEAアプローチを上回っています。
*   **AdaEvolve: Adaptive LLM Driven Zeroth-Order Optimization (2026)**
    *   概要: ゼロ次最適化（勾配を用いない最適化）としてのLLMを用いたプロンプト進化において、資源の無駄を省くための適応型スケジューリング（局所的・大域的な探索強度の調整およびバンディットアルゴリズムによる資源割り当て）を導入した研究です。

## 2. 自動プロンプト最適化（APO）のパラダイムに対する批判的検証や別視点
*EvoPromptの「進化的アプローチ」の対抗馬である「勾配ベース（TextGrad, ProTeGiなど）」に対する重要な検証が含まれています。*

*   **Textual Gradients are a Flawed Metaphor for Automatic Prompt Optimization (2025)**
    *   概要: 自動プロンプト最適化（APO）において広く用いられている「テキストによる勾配（Textual Gradients）」というメタファーが、実際にはその挙動を正確に説明できておらず欠陥があるという実験的検証を行った論文です。これを読むことで、EvoPromptのような「進化的アルゴリズム」がなぜ勾配ベースより有効になり得るのか、別の視点から理解を深めることができます。

## 3. テキストから「実行可能コード / エージェント」への進化アーキテクチャ
*プロンプト（テキスト）だけではなく、プロンプトを実行するサブエージェント自体の進化を扱うアプローチです。*

*   **AgentFactory: A Self-Evolving Framework Through Executable Subagent Accumulation and Reuse (2026)**
    *   概要: LLMエージェントの自己進化において、成功した経験を単なるテキストプロンプトとして保存するのではなく、「実行可能なPythonのサブエージェントコード」として保持し、実行フィードバックに基づいて持続的に精製していくパラダイムを提案しています。テキスト最適化からコード最適化への橋渡しとなる研究です。
