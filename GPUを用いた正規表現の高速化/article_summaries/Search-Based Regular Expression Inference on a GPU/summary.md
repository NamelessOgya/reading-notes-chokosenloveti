# Search-Based Regular Expression Inference on a GPU

## 背景
従来の正規表現推論（Regular Expression Inference, REI）は、正解となる文字列（Positive examples）と拒否すべき文字列（Negative examples）から、それらを正しく分類し、かつ「コスト指標に対して最小」となるような正規表現を合成する問題（Program Synthesis / Supervised Machine Learningの一種）です。しかし、既存のREIアプローチ（例えばAlphaRegexなど）はCPUを用いたヒューリスティックな探索に留まっており、計算コストが膨大になるという課題がありました。本論文では、「ニューラルネットワーク以外の旧来の機械学習アプローチであっても、GPUアクセラレーションによってCPUの2桁以上に及ぶ高速化が可能か？」という問いを立て、正確で最小なREIアルゴリズムのGPU並列化を試みています。

## 手法
著者らは **PaRESy** (Parallel Regular Expression Synthesiser) と呼ばれるアルゴリズムを提案しました。このアルゴリズムのコアとなる技術的本質は、探索空間全体を「メモリ上で連続したビットベクタ（Bitvectors）の行列表現」として簡略化・圧縮することです。

具体的には、
- 正規表現を関係する文字集合のアプローチではなく、関数 $r : \Sigma^* \to \{0, 1\}$ として見立てて正規言語の探索に落とし込んでいます。
- PosとNegの部分文字列閉包（Infix-closure）である $\mathsf{ic}(P \cup N) \to \{0, 1\}$ を探索対象とし、ビットベクタ表現と行列演算を用いたデータ並列設計にしています。

この Infix-closure を用いる手法は、小さな正規表現からより大きな正規表現をボトムアップで合成していく過程において、データの移動とデータ依存による分岐（Branching）を最小化できるため、並列性を非常に高く引き出すことができ、GPUにおけるスレッド・ワープベースの並列処理アーキテクチャに完全に適合する設計となっています。GPUへの実装ではNvidiaのCUDAプラットフォームを利用しています。

## 結果
Google Colab Pro 上の GPU (Nvidia A100) および CPU (Intel Xeon)、ならびに Laptop (MacBook Pro) 環境を用いて、既存の OCaml 制 SOTA（AlphaRegex）と、提案手法である PaRESy の GPU版/CPU版 の性能比較が行われました。
