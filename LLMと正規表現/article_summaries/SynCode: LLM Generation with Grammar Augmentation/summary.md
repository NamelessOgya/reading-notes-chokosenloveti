# SynCode: LLM Generation with Grammar Augmentation

本論文は、文脈自由文法（CFG）に基づくDFAマスクストアをオフライン構築し、LLMのデコード中に文法的に有効なトークンのみを許可する「SynCode」フレームワークを提案した研究である。

---

## 背景

LLMによるコード生成は、関数呼び出しの引数型不一致・括弧の対応ミス・予約語の誤用など、構文エラーを頻繁に含む。特にPython・Go・JSONなどの言語では、小さな文法違反が実行時エラーを引き起こすため、出力の信頼性が重要課題となる。

既存の制約付きデコード手法（例：Outlines/Guidance）は正規表現ベースのFSMによる制約をサポートするが、プログラミング言語の文法全体を表現するCFG（文脈自由文法）への対応は困難だった。CFGはFSMよりも表現力が高い（再帰的なネスト構造を記述できる）一方で、対応するプッシュダウンオートマトン（PDA）の状態空間が無限大になりうるため、事前計算が困難という問題がある。

---

## 手法

### 1. DFAマスクストアのオフライン構築

SynCodeの核心はCFGに基づくDFAマスクストアの事前構築にある。

- **字句解析器（Lexer）とDFA**: プログラミング言語のCFGを字句クラス（数値リテラル・識別子・キーワード等）に分解し、各字句クラスに対してDFAを構築する。
- **インタラクティブラーナー（Earleyパーサベース）**: CFGに対してEarleyパーサを用いて、任意の中間文字列プレフィックスが文法的に有効な継続を持つかを判定する。
- **DFAマスクストア**: 各文法状態（Earleyパーサのアイテムセット）に対して、次に有効な字句クラスを事前計算してマスクストアとして格納する。

$$\text{MaskStore}[s] = \{t \in V \mid t \text{ が文法状態 } s \text{ から有効な継続に対応する}\}$$

### 2. 健全性と完全性の保証

- **健全性（Soundness）**: 生成されるトークン列は常に対象CFGの部分語（文法的に有効なプレフィックス）のみとなる。
- **完全性（Completeness）**: 対象CFGで生成可能なすべての文字列が（理論上）生成候補として現れる。

### 3. Hugging Face統合

SynCodeはHugging FaceのLLMと直接統合可能で、```generate_with_syncode()``` APIを通じて任意のCFGを指定した制約付き生成が行える。

## 結果

### コード生成における文法エラー削減（主要結果）

| 言語 | ベースライン（シンタックスエラー率） | SynCode適用後 | エラー削減率 |
| :--- | :---: | :---: | :---: |
| Python | 高（複数モデルで10〜30%程度） | ほぼ0% | **96.07%削減** |
| Go | 高 | ほぼ0% | **96.07%削減** |
| JSON | 高 | 0% | **100%削減** |

### 生成速度のオーバーヘッド

| 言語 | 追加オーバーヘッド（対非制約生成） |
| :--- | :---: |
| JSON | 約10% |
| Python | 約10〜30% |
| Go | 約10〜30% |

SynCodeはDFAマスクストアを完全にオフラインで構築するため、推論時のオーバーヘッドを最小化しつつ文法保証を実現する。

## chokosenlovetiのメモ

### GPUを用いた正規表現高速化との関係

SynCodeは各プログラミング言語の字句クラスに対してDFAを構築する点で、GPUを用いた正規表現高速化研究との直接的な接点がある。

- **DFAマスクストアの並列検索**: 各バッチ内の複数シーケンスが異なる文法状態にあるとき、異なるDFAを同時に参照する必要がある。これはGPU上でのマルチDFA正規表現マッチング（Yu (2013)のマルチDFA手法）と同一の問題構造である。
- **ウォープ発散の可能性**: バッチ内のシーケンスがそれぞれ異なる文法状態（DFA状態）にある場合、GPU上でマスク計算を実行するとウォープ発散が発生する。DFAの遷移テーブルをGPUフレンドリーに設計する手法（E-DFAのようなアライメント技術）が応用できる。
