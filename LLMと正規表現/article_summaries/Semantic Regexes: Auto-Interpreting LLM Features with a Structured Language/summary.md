# Semantic Regexes: Auto-Interpreting LLM Features with a Structured Language

本論文は、スパースオートエンコーダ（SAE）で得られたLLM内部特徴量を記述するための構造化言語「Semantic Regexes（意味的正規表現）」を提案した研究である。正規表現の形式的な表現力をLLMの解釈可能性研究に適用した。

---

## 背景

**機械的解釈可能性（Mechanistic Interpretability）** の研究では、LLMの内部表現（ニューロンや隠れ状態）がどのような意味的概念に対応するかを理解しようとする。

近年の主要なアプローチは **スパースオートエンコーダ（Sparse Autoencoder: SAE）** であり、LLMの密な活性化ベクトルを疎な（スパースな）単義的な特徴量に分解する：

$$h = \sum_{i} f_i \cdot d_i, \quad \text{s.t.} \quad \|f\|_0 \ll \dim(h)$$

ここで $d_i$ は「特徴方向ベクトル」、$f_i$ はその活性化スコア。各 $d_i$ が理想的には単一の意味的概念（例：「フランス語の動詞」「コードのインデント」）に対応するよう学習される。

### 問題点：自然言語による特徴量説明の限界

各特徴量 $d_i$ が何を表すかを説明するために、従来の自動解釈手法は **自然言語の短い文**（例：「フランス語のテキストに関連する特徴」）を生成する。しかしこのアプローチには：

1. **曖昧さ**: 「フランス語」は文字列か？言語か？文化的文脈か？が不明確
2. **一貫性のなさ**: 同じ特徴量を異なる手法で説明すると異なる記述になる
3. **定量化不能**: 特徴量の「複雑さ」や層間の変化を数値化できない

---

## 手法

### 1. Semantic Regexesの設計

意味的正規表現は、通常の文字列パターン（`\d+` など）とは異なり、**言語的・意味的なパターン**を記述する構造化言語である。

#### プリミティブ（Primitives）

| プリミティブ | 記述 | 例 |
| :--- | :--- | :--- |
| `<pos:POS>` | 品詞タグ | `<pos:VERB>` |
| `<sem:CONCEPT>` | 意味的概念 | `<sem:NUMBER>`, `<sem:LOCATION>` |
| `<lang:LANG>` | 言語 | `<lang:French>` |
| `<topic:TOPIC>` | トピック | `<topic:mathematics>` |
| `<regex:PATTERN>` | 通常の正規表現 | `<regex:\d+>` |

#### 修飾子（Modifiers）

| 修飾子 | 機能 |
| :--- | :--- |
| `CONTEXT(...)` | 文脈条件（前後のトークンに条件） |
| `NEGATE(...)` | 否定 |
| `AND(...)` | 複数条件のAND |
| `OR(...)` | 複数条件のOR |
| `QUANTIFY(...)` | 出現回数の条件 |

#### 記述例

```
# 特徴量の説明例
"フランス語の動詞が主語の後に現れるパターン"
→ CONTEXT(<lang:French>, <pos:VERB>, before=<pos:NOUN>)

"コードブロック内の数値リテラル"
→ CONTEXT(<regex:\d+>, topic=<topic:programming>)
```

### 2. 自動生成パイプライン

1. SAEで特徴量 $d_i$ を抽出する。
2. $d_i$ が高く活性化するトークン列（Top-K）を収集する。
3. LLMに対してTop-Kトークン列を提示し、意味的正規表現の文法に従って特徴量を記述させる。
4. 生成された意味的正規表現で実際のテキストをスキャンし、特徴量の予測精度を検証する。

---

## 結果

### 精度比較（自然言語説明 vs. 意味的正規表現）

| 評価指標 | 自然言語説明 | 意味的正規表現 |
| :--- | :---: | :---: |
| 特徴量活性化予測精度（F1） | 0.71 | **0.73** |
| 記述の一貫性（同じ特徴量への反復適用） | 低 | **高** |
| 特徴量の「複雑さ」の定量化 | 不可 | **可** |
| 人間のメンタルモデル構築（ユーザスタディ） | 中 | **高** |

### 層間の特徴量複雑性分析（新規分析）

意味的正規表現により、各Transformer層における特徴量の複雑さを定量化できる：

- **浅い層（1〜5層）**: 単純なプリミティブ（品詞・文字パターン）が多い
- **中間層（6〜15層）**: CONTEXTやANDによる複合条件が増加
- **深い層（16〜32層）**: 高次の意味的概念（トピック・文体）が支配的

この定量的分析は自然言語説明では実現不可能だった新しい知見である。

## chokosenlovetiのメモ

### GPU正規表現高速化との関係

本論文で提案される意味的正規表現は、形式的な文法をLLM解釈可能性に適用した研究であり、GPU正規表現マッチングとは異なる文脈での正規表現の活用を示している。しかし接点は存在する。

- **意味的正規表現の実行**: 大量のテキストコーパスに対して意味的正規表現をスキャンする操作は、高スループットのパターンマッチングを必要とする。GPU上での正規表現マッチング技術（特にNFAベース）が、意味的正規表現の並列評価に直接応用できる。
- **LLM特徴量のリアルタイム監視**: LLM推論中に特定の特徴量をリアルタイムに意味的正規表現でスキャンするシステムは、GPUの正規表現高速化（iNFAnt・Virtual NFA等）の技術基盤の上に構築できる可能性がある。
- **正規表現によるLLM制御**: 意味的正規表現で記述された特徴量パターンを検出し、LLMの生成を制御するシステムは、制約付きデコードと意味的正規表現の融合として、正規表現高速化を活用できる新しい応用領域である。
