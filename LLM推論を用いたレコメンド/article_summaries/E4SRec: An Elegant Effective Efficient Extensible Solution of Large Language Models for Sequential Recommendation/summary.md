# E4SRec: An Elegant Effective Efficient Extensible Solution of Large Language Models for Sequential Recommendation

## 背景
大規模言語モデル（LLMs）の発展により、レコメンドシステムへの応用が期待されている。既存のアプローチの多くは、レコメンドタスクを自然言語生成タスク（アイテム名や評価の直接生成）に変換してLLMをファインチューニングする手法を採用している。

**代表的な既存アプローチの例（自然言語生成ベース）**

| 出版年 | 論文名 | 手法概要 |
| :--- | :--- | :--- |
| 2022 | **Recommendation as Language Processing (RLP): A Unified Pretrain, Personalized Prompt & Predict Paradigm (P5)** | シーケンシャル推薦、評価予測、説明生成など様々なレコメンドタスクを共通のテキストプロンプト形式に統一し、T5などの言語モデルに自然言語のテキストとしてアイテムや予測を出力させる手法。 |
| 2023 | **TALLRec: An Effective and Efficient Tuning Framework to Align Large Language Model with Recommendation** | LLaMAなどのLLMに対して、ユーザーの過去の行動履歴を文章として入力し、ターゲットアイテムを好むかどうかを言語（例: Yes / No）として直接出力するよう、LoRAを用いて効率的に指示チューニング（Instruction Tuning）を行う手法。 |
| 2023 | **Recommendation as Instruction Following: A Large Language Model Empowered Recommendation Approach (InstructRec)** | レコメンドを「指示に従うタスク」として捉え、ユーザーの好みや推薦の意図を自然言語の指示としてLLMに与えることで、条件に合致する推薦アイテムリストを直接テキストとして生成させる手法。 |
| 2023 | **GenRec: Large Language Model for Generative Recommendation** | 大規模言語モデルの優れたテキスト生成能力を活かし、プロンプトに埋め込まれたテキスト情報から、次にユーザーがクリック/購入するであろう「アイテム名」などの文字列を直接生成させる手法。 |

しかし、この手法には以下の課題がある：
1. 大量の同質アイテムが存在する推薦システムにおいて、セマンティック情報（テキスト等）に依存しすぎると、テキスト情報が乏しい場合に精度が劣化する。一方、産業界で広く使われるコラボラティブフィルタリング等のIDベースの手法で用いられる「アイテムID」のモデリングにLLMは適していない。
2. LLMの語彙全体からテキストを生成するため、候補リストにないアイテムを出力する「範囲外生成（Out-of-range results）」が起こりやすい。
3. 一度に1つの結果しか生成できないため、推論効率が極めて低く、実用的なマッチングタスク等には適さない。

## 手法
これらの課題を解決するため、アイテムIDのみをLLMに入力・処理させ、効率的に推論できるフレームワーク「**E4SRec** (Elegant Effective Efficient Extensible solution for Sequential Recommendation)」を提案した。

アプローチの主要な構成要素は以下の通りである：

1. **ID Injection (ID注入)**
   - 既存のシーケンシャル推薦モデル（本論文ではSASRecを使用）を事前学習させ、そのモデルからアイテムIDエンベディング（$\mathbf{E} \in \mathbb{R}^{N \times d_s}$）を抽出する。これにより、セマンティック情報を持たないIDのみでコラボラティブ（協調的）な情報をLLMに与える。
2. **Backbone Tuning (LLMのインストラクションチューニング)**
   - LLM（LLaMa2-13B）がフォーマットに厳密に従えるよう、Alpacaテンプレートを用いてLoRA（Parameter-Efficient Fine-Tuning）でインストラクションチューニングを行う。
3. **E4SRec Model Training (E4SRecの学習)**
   - **入力層**: 入力はテキストとIDベクトルのハイブリッドである。Alpacaテンプレートに則った形式的な指示文（テキスト）はWordエンべディングとして入力される一方、ユーザーの過去の行動履歴（アイテム系列）はアイテム名などの文字列としてではなく、LLMのトークン次元に合わせるための線形射影 $\mathbf{W}^{Input} \in \mathbb{R}^{d_s \times d_k}$ を用いて変換したアイテムIDエンベディングとして、通常のプロンプトトークンと結合（Inject）される。
   - **LLM層**: インストラクションチューニング済みのLLM層に対し、対象データセット特有のパーソナライゼーションを学習させるためのLoRAアダプター（$\Theta$）を追加する。
   - **予測層**: テキストを出力（推論）することは一切しない。言語生成タスクのタスク定義（全ボキャブラリからの逐次テキスト生成）を廃止し、LLMの出力ベクトルに対しアイテム線形射影 $\mathbf{W}^{Output} \in \mathbb{R}^{d_k \times N}$ を適用して、全候補アイテム $N$ 個にわたる結合確率分布（予測スコアベクトル）を一括で出力する。損失関数はクロスエントロピー誤差を用いる：
     $$ \mathcal{L} = -\sum_{i=1}^N y_i \log(\hat{y}_i) $$
4. **Deploy (拡張性と推論)**
   - 推論時にはLLMの通常の「Softmaxを用いた逐次トークン生成」を廃止し、LLMが出力した1つのベクトルと $\mathbf{W}^{Output}$ との最近傍探索（Nearest neighbor search）へと簡略化することで、従来の手法と同等の推論効率を維持している。
