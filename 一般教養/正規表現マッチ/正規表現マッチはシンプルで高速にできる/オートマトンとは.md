# オートマトン（Automaton）とは何か？

「オートマトン（複数形: Automata）」とは、計算機科学や数学において**「入力に応じて内部の『状態』を変化させていく仮想的な機械（数学的モデル）」**のことを指します。

先ほどの記事「正規表現マッチはシンプルで高速にできる」を深く理解するために必要な、オートマトンの基本的な考え方と、DFA・NFAの違いについて解説します。

---

## 1. オートマトンの基本（日常にある具体例）

「入力によって状態が変わる機械」というと難しく聞こえますが、実は私たちの身の回りにあふれています。最もわかりやすい例が**「自動ドア（または改札機）」**です。

例えば、単純なスイング式の自動ドア（または駅の改札）を「オートマトン」としてモデル化すると、次のようになります。

1. **状態（State）**: ドアには「しまっている（閉）」と「あいている（開）」の**2つの状態**しかありません。
2. **入力（Input）**: センサーが人を「感知した」または「感知していない」。
3. **遷移（Transition）**: 
   - 現在「閉」で、人を「感知」したら → 「開」に状態遷移する。
   - 現在「開」で、人を「感知していない」なら → 「閉」に状態遷移する。

このように、「今の状態は何か？」「何が入力されたか？」によって「次にどの状態になるか？」を矢印で結んで表現したものがオートマトンです。

---

## 2. 正規表現とオートマトンの関係

記事で登場したオートマトンは文字列の「マッチング」に使われていました。文字列の1文字1文字が、上記の「入力」にあたります。

記事内の `a(bb)+a` という正規表現にマッチするか確認する有限オートマトンを思い出してください。

- **初期状態（スタート）**: 単語の1文字目を読む前の状態。
- **遷移（矢印）**: 文字列から `a` や `b` といった文字を1文字ずつ読み込み、矢印の条件に合致すれば次の丸（状態）へ進む。
- **受理状態（ゴール）**: 二重丸（◎）。ここにたどり着いた状態で文字列を全て読み終えていれば、「この文字列は正規表現にマッチした（受理された）」と判定する。

もし、読み込んでいる途中で「次の文字に合致する矢印が存在しない」場合、ゴールにたどり着けないため「マッチしなかった（エラー）」となります。これがオートマトンによる文字列検索の仕組みです。

## 3. DFA（決定性）と NFA（非決定性）の違い

先ほどの記事では、特に **DFA** と **NFA** という2つの異なるオートマトンが比較されていました。この違いを理解することが、記事の核心（Thompsonの実装がなぜ速いのか）を理解する鍵です。

### ① DFA（決定性有限オートマトン / Deterministic Finite Automaton）
DFAは、「決定性（迷いがない）」という名前の通り、**「ある状態にいるとき、特定の文字を読み込んだら、次に進む先が必ず1つに決まる」**オートマトンです。

- **メリット**: 実行が爆速。文字を1文字読むごとに、矢印を1本たどるだけなので、どれだけ長い文字列でも「文字数と同じ回数」矢印をたどるだけで結果がわかります。
- **デメリット**: 複雑な正規表現になると、「迷わなくて済むように」あらかじめすべてのパターンを網羅した状態（丸）を用意しなければならず、構築するのに時間がかかったり、メモリ（状態の数）が爆発的に増えたりするリスクがあります。

### ② NFA（非決定性有限オートマトン / Non-deterministic Finite Automaton）
NFAは、「非決定性」という名前の通り、**「ある状態にいるとき、同じ文字を読んでも進む先の候補が『複数』ある（分岐する）」**オートマトンです。また、**「何も文字を読んでいないのに勝手に次の状態へワープする矢印（空遷移）」**も許されます。

- **メリット**: 正規表現からオートマトンを自動生成（コンパイル）するのが非常に簡単で、状態の数も正規表現の長さに比例して小さく済みます。（記事内の図で、NFAがブロックを組み合わせるように作れると説明されていた通りです）。
- **デメリット**: 分岐（選択肢）があるため、「どの矢印を選ぶのが正解だったか？」を文字列を最後まで読まないと判断できません。

---

## 4. 記事の核心：「NFAの分岐をどう処理するか？」

Perl や Python などが採用している旧来の実装（**バックトラッキング**）は、NFAに分岐があったとき、「とりあえず道Aに行ってみて、行き止まりだったら戻ってきて道Bを試す」というやり方をします。
これが、記事で「病的な正規表現（例： `a?nan` ）」を与えられた時に何年計算しても終わらなくなる原因です（すべての分岐の組み合わせを試そうとして文字通り指数関数的に計算爆発を起こすためです）。

一方で Ken Thompson が考案したアルゴリズムは、**「道Aに行く自分と、道Bに行く自分を同時に分裂させて、並行して進める（マルチステート・アプローチ）」**という方法をとります。
これにより、「間違えたら戻る」という無駄が発生せず、一度文字列を舐めるだけで必ずマッチ判定が終わるため、どんなにひどい正規表現を渡されても **一貫して爆速（線形時間）で処理できる** のです。

### まとめ
* **オートマトン**とは、入力に応じて状態が変わることを定義した「状態遷移のルールブック（図）」のこと。
* **正規表現（テキスト検索）**は、オートマトンという数学の図に変換することで、素早く機械的に処理できる。
* **Thompson NFA手法**は、「後戻り（バックトラッキング）」をしない賢いアルゴリズムを使うことで、現代の多くの言語が抱えている「重い正規表現だとフリーズする」という問題を完全に回避している。
