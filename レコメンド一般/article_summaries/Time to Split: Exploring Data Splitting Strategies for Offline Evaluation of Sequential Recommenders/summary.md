# Time to Split: Exploring Data Splitting Strategies for Offline Evaluation of Sequential Recommenders

- **著者**: Danil Gusak, Anna Volodkevich, Anton Klenitskiy, Alexey Vasilev, Evgeny Frolov
- **発表**: ACM RecSys 2025
- **arXiv**: [2507.16289](https://arxiv.org/abs/2507.16289)
- **コード**: https://github.com/monkey0head/time-to-split

---

## 背景

逐次推薦システム（SRS）はnext-item prediction（NIP）タスクで広く使われているが、オフライン評価のためのデータ分割戦略は依然として未成熟である。

**問題の核心**：
- 研究の77.3%（RecSys/SIGIR/CIKM 2022–2024の75論文中）が **Leave-One-Out（LOO）** 分割を採用しているが、LOOはグローバルタイムラインを無視し、**時間的リーク（Temporal Leakage）** を引き起こす。訓練データに未来のインタラクションが混入し、現実のデプロイシナリオとかけ離れた長いテスト期間をもたらす。
- **Global Temporal Split（GTS）** はこの問題を解決するが、NIPタスクへの適用方法（ターゲットアイテム選択・バリデーション構築）が標準化されていない。
- 既存フレームワーク（Cornac, RecBole等）もGTS+NIPに対応したsplitをほとんど提供していない（Table 1）。

本論文はこのギャップを埋めるため、SRS向けに特化したGTS変種を体系的に比較し、再現性のある評価プロトコルを確立することを目的とする。

---

## 手法

### 分割戦略の整理

#### 1. Leave-One-Out (LOO)
各ユーザの**最後のインタラクション**をテスト、**最後から2番目**をバリデーションとする。
- **長所**: NIPタスクと整合、学習データが多い、テストユーザ100%
- **短所**: グローバルタイムラインを無視 → 時間的リーク、テスト期間が非現実的に長い

#### 2. Global Temporal Split (GTS)
インタラクションデータの分位数 $q$ に基づくグローバル時刻 $T_{\text{test}}$ を定め、$T_{\text{test}}$ 以降のインタラクションをホールドアウトとする。

**GTSのターゲット選択オプション**（Figure 1c相当）：

| ターゲット | 説明 | 特徴 |
|-----------|------|------|
| **All** | ユーザの全ホールドアウトをground-truthセットとして利用 | タスクミスマッチ |
| **Last** | ホールドアウト中の最後のインタラクションをターゲット | LOO+GTSの直感的組み合わせ |
| **First** | ホールドアウト中の最初のインタラクションをターゲット | セッション境界バイアスあり |
| **Successive (Sucv.)** | ホールドアウトの各インタラクションを個別ターゲットとして評価 | 最も現実的・計算コスト高 |
| **Random** | ホールドアウトからランダムに1つ選択 | Successiveの代替・再現性要注意 |

**Successive評価スキーム**（Figure 2相当）：
ホールドアウト長を $n$ とすると、$n$ 個の個別評価を行い、ユーザ内で事前平均化する。

$$\text{Metric}_{user} = \frac{1}{n} \sum_{k=1}^{n} \text{Metric}(target_k)$$

#### 3. GTSのバリデーション戦略（Figure 3相当）

| 戦略 | 説明 | 特徴 |
|------|------|------|
| **GT（Global Temporal）** | $T_{\text{val}}$ 以降をバリデーション | リーク無し、学習データ減少 |
| **LTI（Last Training Item）** | $T_{\text{test}}$ 前の最後のインタラクションをバリデーションターゲット | 全ユーザカバー、期間が非現実的 |
| **UB（User-Based）** | ランダムなユーザサブセットの全履歴をバリデーション | 学習データ減少を抑制、期間が非現実的 |

### Figure 4: 時間ギャップ分布（Zvukデータセット）

![Delta Hist](./images/delta_hist.png)

**考察（時間ギャップ $\delta$ に対する筆者の分析と見解）**:

1. **First ターゲットにおける「セッション境界（Session-Boundary）バイアス」の発生**
   - Table 3 を見ると、First ターゲット（グローバル分割点 $T_{\text{test}}$ 直後のインタラクション）の時間ギャップ $\delta$ が、他のターゲットやデータセット全体の平均（Full Data）と比べて異常に大きな値を示しています（例: ML-20M では全体のメディアンが 11 秒であるのに対し、First では約 244 日に達します）。
   - 筆者は、グローバルな分割点がユーザの非活動期間（セッション間）に位置しやすいため、直後の First アイテムが「新しいセッションの最初のアイテム」になりやすいと指摘しています。これは「セッション内の継続的な推薦」ではなく「新規セッションの立ち上げ予測」というタスクの変質（バイアス）をもたらすため、**First ターゲットをオフライン評価に使用することは非推奨**と結論づけています。

2. **Amazon Beauty / Sports におけるホールドアウトの短さの影響**
   - シーケンス長が極めて短いデータセット（Beauty や Sports など）では、ホールドアウト期間内のインタラクション数が 1〜2 個程度と非常に少なくなります。
   - このため、First, Last, Random の各ターゲットが結果的に同一のアイテムを指す確率が高まり、Last や Random でも First と同様のセッション境界バイアスの影響を部分的に受けてギャップが拡大する現象がみられます。

3. **First 以外のターゲット（Last, Random, Successive）の健全性**
   - First ターゲットを除く他すべてのターゲット（Last, Random, Successive）のインタラクション間隔は、データセット全体の自然なインタラクション間隔分布（Full Data）と非常に良く一致しています。
   - したがって、First さえ避ければ、時間的間隔の観点から不自然なバイアスを伴わない**健全かつ現実的なオフライン評価が可能である**と筆者らは評価しています。

## 総括

本論文の実験から以下の結論が得られた：

1. **推奨される分割戦略**: GTS with **Last**・**Random**・**Successive** ターゲットが適切。Last・Randomは計算コストの高いSuccessiveの合理的代替となる。
2. **非推奨**: LOO（時間的リーク）、GTS First（セッション境界バイアス）、GTS All（タスクミスマッチ）
3. **バリデーション**: Global Temporal（GT）バリデーションが最も信頼性が高く、テスト指標との一貫性を保証する。
4. **再学習**: GT・UBバリデーション使用時は、訓練+バリデーションデータ合算での再学習が最終テスト性能の向上に重要。
